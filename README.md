## نظرة عامة

**نسق** هو نظام ذكاء اصطناعي متخصص في دعم التصميم التعليمي، طُوِّر ضمن **هاكاثون ذكاءثون 2026** في جامعة الأميرة نورة بنت عبدالرحمن. تستهدف المنصة أعضاء هيئة التدريس وتمكّنهم من تحويل نواتج التعلم إلى أهداف إجرائية سلوكية وأنشطة تعليمية متنوعة وسلالم تقدير جاهزة للاستخدام — كل ذلك في دقائق بدلاً من ساعات.

> **الرابط المباشر:** [nasaqai-4jdv7yai.manus.space](https://nasaqai-4jdv7yai.manus.space)

## المشكلة والفرصة

يواجه أعضاء هيئة التدريس صعوبة في تكييف المقررات واختيار الأنشطة وسلالم التقدير التي تخدم نواتج التعلم مع بداية كل فصل دراسي. **نسق** يقدم حلاً ذكياً يعتمد على تقارير وتحليلات المتعلمين لاقتراح الأنشطة وسلالم التقدير المناسبة والتي حققت نواتج التعلم المأمولة سابقاً.

| التحدي | الحل في نسق |
| --- | --- |
| صياغة أهداف إجرائية وفق تصنيف بلوم | توليد تلقائي بنموذج نهى |
| اختيار أنشطة تتناسب مع المستوى الأكاديمي | اقتراح نشاطين متنوعين لكل ناتج |
| بناء سلالم تقدير (Rubrics) متسقة | سلم تقدير جاهز لكل نشاط |
| تصدير الوثائق للاستخدام الفوري | تصدير Word بضغطة واحدة |

## لماذا نسق؟

الاسم **نسق** مستوحى من مفهوم **التنسيق والتكامل**؛ فالمنصة لا تعمل كأداة معزولة بل تتكامل مع منظومة التعليم الرقمي بأكملها:

- **تكامل مع أنظمة LMS:** مصمّمة للعمل جنباً إلى جنب مع أنظمة إدارة التعلم (Blackboard, Moodle, Canvas) لاستيراد بيانات المتعلمين وتحليل أدائهم

- **تنسيق شامل:** تنسّق بين نواتج التعلم والأهداف الإجرائية والأنشطة وسلالم التقدير في منظومة تعليمية متكاملة

- **قرارات مبنية على بيانات:** تستخدم بيانات أداء المتعلمين الفعلية لاقتراح الأنشطة الأكثر فاعلية

## الميزات الرئيسية

### 1. إنشاء المقررات

رفع الخطة الدراسية أو إدخال بيانات المقرر يدوياً (الاسم، الرمز، المستوى، عدد النواتج).

### 2. توليد الأهداف الإجرائية

يحلل نموذج نهى نواتج التعلم ويولّد أهدافاً إجرائية سلوكية مصنّفة وفق **تصنيف بلوم** (تذكر، فهم، تطبيق، تحليل، تقييم، إبداع).

### 3. توليد الأنشطة التعليمية

لكل ناتج تعلم، يقترح النظام **نشاطين تعليميين متنوعين** (مشاريع، نقاشات، اختبارات، مهام تطبيقية) مع وصف تفصيلي لكل نشاط.

### 4. سلالم التقدير (Rubrics)

لكل نشاط، يُنشئ النظام **سلم تقدير** متكامل بمعايير واضحة ومستويات أداء محددة.

### 5. التصدير

تصدير الخطة التعليمية الكاملة بصيغة **Word (.docx)** جاهزة للتقديم والاستخدام الفوري.

### 6. العرض الهرمي

عرض تفاعلي يُظهر العلاقة الهرمية بين: نواتج التعلم ← الأهداف الإجرائية ← الأنشطة ← سلالم التقدير.

## التثبيت والتشغيل

### المتطلبات الأساسية

- Node.js 22+

- pnpm

- قاعدة بيانات MySQL (أو TiDB)

- مفتاح API لنموذج نهى من منصة علم

### خطوات التثبيت

```bash
# 1. استنساخ المستودع
git clone <repository-url>
cd nasq-prototype

# 2. تثبيت الاعتماديات
pnpm install

# 3. إعداد متغيرات البيئة
cp .env.example .env
# أضف القيم المطلوبة في ملف .env

# 4. تهيئة قاعدة البيانات
pnpm db:push

# 5. تشغيل الخادم
pnpm dev
```

### متغيرات البيئة المطلوبة

| المتغير | الوصف |
| --- | --- |
| `DATABASE_URL` | رابط اتصال قاعدة بيانات MySQL |
| `JWT_SECRET` | مفتاح توقيع الجلسة |
| `NAHYA_API_KEY` | مفتاح API لنموذج نهى (منصة علم) |
| `NAHYA_API_BASE` | رابط API نهى من منصة علم |
| `VITE_APP_ID` | معرّف تطبيق Manus OAuth |
| `OAUTH_SERVER_URL` | رابط خادم Manus OAuth |

## نموذج نهى

**نهى 2.0** هو نموذج لغوي كبير (LLM) طوّرته **شركة علم**، متخصص في اللغة العربية والمحتوى الأكاديمي. يُستخدم في نسق لثلاث مهام رئيسية:

### 1. توليد وصف المقرر

```
المدخل : اسم المقرر + المستوى الأكاديمي + نواتج التعلم
المخرج : وصف أكاديمي شامل للمقرر
```

### 2. تحويل نواتج التعلم إلى أهداف إجرائية

```
المدخل : ناتج التعلم + مستوى الطلاب
المخرج : 3-5 أهداف إجرائية سلوكية مصنّفة وفق تصنيف بلوم
         (معرفي / مهاري / وجداني)
```

### 3. توليد الأنشطة وسلالم التقدير

```
المدخل : الأهداف الإجرائية + السياق الأكاديمي
المخرج : نشاطان تعليميان متنوعان (من 6 أنواع: مشروع، نقاش،
          اختبار، عملي، عرض، بحث) + سلم تقدير لكل نشاط
```

يتم استدعاء النموذج عبر ملف `server/_core/nahya.ts` الذي يوفر واجهة موحّدة متوافقة مع معيار OpenAI API.
